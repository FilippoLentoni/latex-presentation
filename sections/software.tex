\section{Traditional Software Evaluation}

% ---------------- Slide 5 ----------------
\begin{frame}{1) Traditional Software Engineering}
\begin{itemize}
  \item Deterministic program: same input $\Rightarrow$ same output
  \item Correctness is (mostly) binary and spec-driven
  \item Failures are reproducible: bugs, regressions, integration issues
\end{itemize}
\end{frame}

% ---------------- Slide 6 ----------------
\begin{frame}{Software evaluation toolbox}
\begin{itemize}
  \item Unit tests, integration tests, end-to-end tests
  \item Contracts, invariants, assertions
  \item Static analysis, type checks, fuzzing
\end{itemize}
\begin{block}{Mental model}
If it fails, the code is wrong (or the spec is).
\end{block}
\end{frame}
